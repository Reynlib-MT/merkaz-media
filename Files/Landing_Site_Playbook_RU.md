# Playbook: одностраничный лендинг (универсальный)

Правила для любого проекта: студия, радиостанция, сервис, продукт, иврит/русский/английский, RTL или LTR.  
Перед работой заполните **бриф проекта** (раздел 1) — остальные разделы применяются с подстановкой своих значений.

---

## 1. Бриф проекта (заполнить для каждого сайта)

Перед работой зафиксировать:

- язык и направление (`lang`, `dir`)
- аудитория
- цель страницы и главное действие (кнопка / форма / play)

**Жёсткое правило для всех проектов:** не выдумывать отзывы, логотипы партнёров, цифры, награды и кейсы без подтверждения.

## 2. Метод копирайта (стиль Timochko)

Полное описание метода — в отдельном файле: **[Timochko_Copy_Method_RU.md](Timochko_Copy_Method_RU.md)**  
Ниже — краткая выжимка для работы над лендингом.

### 2.1. Суть

Лендинг ведёт к **одному действию** короткими блоками: заголовок = мысль, 3–4 буллета, одна кнопка. Мысль формулируется **от читателя**, конкретно, позитивно (что есть — не чего нет).

### 2.2. Голос

Один голос на сайт: личный эксперт («я») или компания/бренд («мы»). Лицо — в одном блоке, не в каждом заголовке.

### 2.3. Заголовок = текст

H1 — что и для кого; H2 — результат или вопрос аудитории; акцент на 1–3 словах; eyebrow — контекст строкой.

### 2.4. Анти-ИИ

Избегать отрицаний («без», «не только», `בלי`, `לא רק`) и шаблонов: «бесплатная консультация», «лучшие», длинные списки в hero. Писать через наличие и конкретику.

### 2.5. Одна мысль — одно место

Главный оффер, первый шаг, доверие, лицо, popup и полная форма — каждый свой угол, без повторов.

### 2.6. CTA

Спокойный тон, ротация формулировок по секциям, кнопка = следующий шаг (не «получить бесплатно»).

## 3. Типографика (масштабируемая система)

### 3.1. Выбор шрифта

- Один семейство на весь сайт, 2–3 начертания (400, 600/700, 800).
- Для иврита: Assistant, Heebo, Rubik и т.п.
- Для кириллицы: Inter, Manrope, Onest и т.п.
- Проверить читаемость цифр в формах и кнопках.

### 3.2. Desktop — рекомендуемая шкала


| Роль          | Размер                     | Начертание                      |
| ------------- | -------------------------- | ------------------------------- |
| body          | **18px**                   | 400–500, line-height 1.5–1.6    |
| h1            | `clamp(36px, 5vw, 64px)`   | 800                             |
| h2            | `clamp(28px, 3.8vw, 48px)` | 800                             |
| h3 / карточки | **20px**                   | 700                             |
| eyebrow       | **16px**                   | 700, letter-spacing чуть шире   |
| буллеты       | 18–19px                    | как body                        |
| кнопки        | **16–17px**                | 700                             |
| поля формы    | **16px**                   | плейсхолдер — приглушённый цвет |


Заголовки: плотный line-height (1.08–1.15).

### 3.3. Mobile — одно правило на `@media (max-width: 768px)`

**Не плодить 480px / 576px для кегля** — типографика переключается на phone одним брейкпоинтом.


| Роль                         | Mobile                    |
| ---------------------------- | ------------------------- |
| h1, h2                       | **34px**                  |
| h3 в карточках / шагах       | **22px**, weight 800      |
| body, буллеты, формы, кнопки | **18px**                  |
| eyebrow                      | **16px** (не увеличивать) |
| акцент в success / имени     | **20px**                  |


Дополнительные брейкпоинты (480px и т.д.) — **только для особых UI** (сложная схема, змейка, не текст).

### 3.4. Переносы и неразрывные пробелы

- Брендовые пары и короткие фразы в заголовках — `nowrap` или  ``.
- Составные имена (App Store, Google Play, название станции) — неразрывный дефис `‑` где нужно.
- **Висячие слова:** JS заменяет последний пробел в текстовом узле на `\u00A0` (кроме форм и `script`/`style`).

### 3.5. Поля ввода

- Подписи — в **placeholder** + `aria-label` (чище визуально).
- Текст в поле: основной цвет; placeholder: `--muted`.
- Телефон в RTL: `direction: ltr; text-align: end` (визуально к краю, цифры не зеркалятся).

## 4. Адаптив: принципы брейкпоинтов

Использовать **2–4 порога**, каждый с одной задачей:


| Уровень        | Типичный max-width | Задача                                             |
| -------------- | ------------------ | -------------------------------------------------- |
| Desktop wide   | —                  | полная сетка, крупный визуал                       |
| Desktop narrow | 1180–1280px        | hero в 1 колонку, если текст + медиа не помещаются |
| Tablet         | 1080px             | сетки 3→1, карточки 2 в ряд                        |
| Phone          | 768px              | меню-бургер, mobile-типографика, упрощение декора  |


**Правила:**

- tablet и phone **не сливать** в один брейкпоинт, если на планшете ещё 2 колонки.
- при нехватке места в hero: сначала **уменьшить gap/padding**, потом менять пропорции колонок; **не сжимать медиа** ради текста через жёсткий `minmax` на текстовой колонке.
- на RTL full-bleed полосы: физический `margin-left` + `width: 104%`, не полагаться на `margin-inline: -1%`.

## 5. Визуальная система

### 5.1. Токены (задать под бренд)

```css
:root {
  --ink:       /* основной текст */;
  --muted:     /* вторичный текст */;
  --line:      /* границы */;
  --bg:        /* фон страницы */;
  --accent:    /* ссылки, hl-accent */;
  --cta:       /* primary кнопка */;
  --radius:    16–24px;
  --container: 1200–1400px;
  --pad:       clamp(20px, 4vw, 40px);
}
```

Палитру брать из логотипа. CTA может быть градиентом, текст — нейтральным.

### 5.2. Паттерны layout (модули — выбирать нужные)


| Модуль                     | Когда использовать                        |
| -------------------------- | ----------------------------------------- |
| Hero split (текст + медиа) | продукт, студия, станция с визуалом       |
| Hero centered              | один сильный оффер, мало медиа            |
| Logo / partner marquee     | B2B, спонсоры                             |
| Карточки 3 в ряд           | услуги, программы, тарифы                 |
| Тёмный «остров»            | выделить блок (deep-dive, contact, плеер) |
| Process / steps            | путь клиента или «как это работает»       |
| Popup micro-form           | быстрый лид с рекламы                     |
| Полная форма внизу         | квалифицированные лиды                    |


Радиостанции чаще: hero + live/player + расписание + подкасты + о станции + контакты — **не обязаны** иметь portfolio и «6 шагов process».

### 5.3. Медиа

- Реальные фото/скриншоты/обложки, не стоковый «tech abstract», если бренд живой.
- Hero-коллаж: не уменьшать ради текста; на узком экране — стек, медиа по центру с `max-width`.
- Логотип в header: читаемая высота (ориентир **48–60px** desktop, **40–48px** mobile), без лишней подложки.

### 5.4. Акценты в тексте

- Hero: градиент на тексте (`.hl`) — один раз на странице.
- Остальные секции: один цвет акцента (`.hl-accent`).
- На тёмном фоне — осветлённый акцент для контраста.

## 6. Формы

### 6.1. Два уровня (рекомендуется)


|         | Popup (micro)                  | Форма внизу (full)         |
| ------- | ------------------------------ | -------------------------- |
| Поля    | 1–2 (имя, телефон)             | + email, тип, комментарий  |
| Триггер | CTA-кнопки по сайту            | ссылка «контакты» в footer |
| Текст   | выгода + срок ответа           | развёрнутый контекст       |
| Кнопка  | следующий шаг (звонок, запись) | отправить / обсудить       |


### 6.2. Тексты popup (шаблон Timochko-micro)

- **h2** — результат: «ответим / перезвоним / откроем доступ» + акцент на сроке или формате.
- **lead** — одна строка доверия, не «заполните поля».
- **submit** — действие: «записаться на звонок», «отправить», «слушать» — без «бесплатная консультация».
- **success** — коротко: «получили» + «свяжемся soon».

### 6.3. UI popup

- Backdrop: тёмный полупрозрачный + лёгкий blur.
- Панель: фон **не белый на белом** — светло-серый градиент; поля **белые** с рамкой и тенью.
- Скругление и `overflow: hidden` на **обёртке панели**, не только на форме.
- Крестик: на стороне «закрытия» для вашего RTL/LTR; без круглой подложки, если мешает.
- Поля в ряд: `grid-template-columns: minmax(0,1fr) minmax(0,1fr)` + `width:100%` на input.
- Закрытие: ×, клик по фону, Escape.

## 7. Навигация и footer

- **4–6 пунктов** максимум; якоря на блоки.
- Sticky header с лёгким blur.
- Один primary CTA в header.
- Footer: логотип, теглайн, дубль навигации, копирайт.

## 8. Ассеты и файлы

```
assets/
  images/     — активные медиа
  logos/
  unused/     — архив, не подключать
```

- Один формат на место (не .png и .jpg дубликаты в проде).
- `loading="lazy"` ниже fold.
- Alt — описательный, на языке сайта.
- Favicon и OG-теги под каждый проект.

---

## 9. Техника (стек без фреймворков)

- `index.html` + `styles.css` + `script.js` — допустимо для лендингов.
- `prefers-reduced-motion` — отключать анимации.
- Бесконечный marquee — JS (rAF), не длинная CSS-animation.
- Формы: success-state на клиенте; интеграция с CRM/почтой — отдельная задача.
- `direction: ltr` точечно: трек marquee, телефон, смешанная латиница в RTL.

## 10. Чеклист перед публикацией

**Контент**

- [ ] Бриф заполнен: язык, аудитория, одно главное действие
- [ ] Нет выдуманных фактов и логотипов
- [ ] Нет шаблонных отрицаний («без», «не только») в новых текстах
- [ ] Одна мысль не повторяется в 3 секциях
- [ ] CTA различаются по блокам

**Типографика и UI**

- [ ] Mobile: h1/h2 = 34px, body = 18px (или своя шкала — но единообразно)
- [ ] Формы: placeholder + aria-label
- [ ] Popup и полная форма согласованы по тону
- [ ] Ключевые фразы не рвутся на висячее слово

**Техника**

- [ ] Все src картинок существуют
- [ ] Работает на phone и tablet
- [ ] Reduced motion учтён
